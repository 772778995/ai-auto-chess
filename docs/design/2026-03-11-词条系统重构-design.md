# 词条系统重构设计

> 将词条系统从数据驱动重构为函数驱动，支持纯函数、状态列表、可序列化

---

## 设计目标

1. **函数驱动**：词条效果以纯函数形式定义，最大灵活性
2. **纯函数**：所有效果函数无副作用，输入 GameState 返回新 GameState
3. **状态列表**：能力值变化通过 statusList 管理，可追溯来源
4. **可序列化**：函数可序列化为字符串，支持 AI 自动生成卡牌
5. **服务端执行**：效果逻辑仅在服务端运行，客户端只渲染结果

---

## 核心概念

### statusList vs currentHealth

| 概念 | 说明 | 示例 |
|------|------|------|
| `statusList` | 能力值变化（攻击力、生命上限、护盾） | +2攻击、+3生命上限、+1护盾 |
| `currentHealth` | 当前血量，战斗中动态变化 | 受到伤害、受到治疗 |

**关键区分**：
- **伤害** → 直接扣 `currentHealth`（即时效果）
- **生命上限变化** → 进入 `statusList`（状态变化）

## 数据结构

### StatusItem（状态项）

```typescript
interface StatusItem {
  attack?: number      // 攻击力加成
  health?: number      // 生命上限加成
  shield?: number      // 护盾层数
  permanent: boolean   // 是否永久保留
  source: string       // 来源描述
}
```

### FollowerTemplate（随从模板）

```typescript
interface FollowerTemplate {
  id: string
  name: string
  description: string
  level: 1 | 2 | 3 | 4 | 5 | 6
  baseAttack: number
  baseHealth: number
  effects: Partial<Record<TriggerTiming, EffectDefinition>>
  equipmentSlots: number
  imageUrl: string
}
```

### FollowerInstance（随从实例）

```typescript
interface FollowerInstance extends Omit<FollowerTemplate, 'effects'> {
  instanceId: string
  ownerId: string
  position: BattlePosition
  currentHealth: number    // 当前血量
  statusList: StatusItem[] // 状态列表
  equipment: string[]
}
```

### EffectDefinition（效果定义）

```typescript
interface EffectDefinition {
  description: string
  execute: EffectFunction
}

type EffectFunction = (ctx: EffectContext) => GameState
```

### EffectContext（执行上下文）

```typescript
interface EffectContext {
  gameState: GameState
  self: FollowerInstance
  tools: EffectTools
  event?: GameEvent
}

interface EffectTools {
  cloneDeep: <T>(obj: T) => T
  getRandomEnemy: (state: GameState, pos?: BattlePosition) => FollowerInstance | null
  getRandomAlly: (state: GameState, pos?: BattlePosition) => FollowerInstance | null
  getAllAllies: (state: GameState) => FollowerInstance[]
  getAllEnemies: (state: GameState) => FollowerInstance[]
  getColumnAllies: (state: GameState, x: number) => FollowerInstance[]
}
```

### TriggerTiming（触发时机）

```typescript
type TriggerTiming =
  | 'onEnter'       // 入场
  | 'onGrowth'      // 成长
  | 'onFirstStrike' // 先手
  | 'onAttack'      // 攻击后
  | 'onHit'         // 受击
  | 'onKill'        // 击杀
  | 'onDeath'       // 死亡（遗言）
  | 'onRoundEnd'    // 回合结束（准备）
```

## 效果函数示例

### 入场伤害

```typescript
effects: {
  onEnter: {
    description: '入场时对随机敌人造成2点伤害',
    execute: (ctx: EffectContext): GameState => {
      const newState = ctx.tools.cloneDeep(ctx.gameState)
      const target = ctx.tools.getRandomEnemy(newState, ctx.self.position)
      if (target) {
        target.currentHealth -= 2
      }
      return newState
    }
  }
}
```

### 永久增益

```typescript
effects: {
  onEnter: {
    description: '入场时所有友方随从获得+2/+2',
    execute: (ctx: EffectContext): GameState => {
      const newState = ctx.tools.cloneDeep(ctx.gameState)
      const allies = ctx.tools.getAllAllies(newState)
      for (const ally of allies) {
        ally.statusList.push({
          attack: 2,
          health: 2,
          permanent: true,
          source: `${ctx.self.name}入场增益`
        })
      }
      return newState
    }
  }
}
```

### 成长效果（永久）

```typescript
effects: {
  onGrowth: {
    description: '成长时永久获得+1/+1',
    execute: (ctx: EffectContext): GameState => {
      const newState = ctx.tools.cloneDeep(ctx.gameState)
      const self = newState.allies.find(f => f.instanceId === ctx.self.instanceId)
      if (self) {
        self.statusList.push({
          attack: 1,
          health: 1,
          permanent: true,
          source: '成长效果'
        })
      }
      return newState
    }
  }
}
```

### 先手效果（临时）

```typescript
effects: {
  onFirstStrike: {
    description: '先手时获得+2攻击',
    execute: (ctx: EffectContext): GameState => {
      const newState = ctx.tools.cloneDeep(ctx.gameState)
      const self = newState.allies.find(f => f.instanceId === ctx.self.instanceId)
      if (self) {
        self.statusList.push({
          attack: 2,
          permanent: false,
          source: '先手效果'
        })
      }
      return newState
    }
  }
}
```

## 工具函数

### 状态计算

```typescript
function getCurrentStats(follower: FollowerInstance) {
  const statusTotal = follower.statusList.reduce(
    (acc, s) => ({
      attack: acc.attack + (s.attack ?? 0),
      health: acc.health + (s.health ?? 0),
      shield: acc.shield + (s.shield ?? 0)
    }),
    { attack: 0, health: 0, shield: 0 }
  )
  
  return {
    attack: follower.baseAttack + statusTotal.attack,
    maxHealth: follower.baseHealth + statusTotal.health,
    shield: statusTotal.shield,
    currentHealth: follower.currentHealth
  }
}
```

### 清除临时状态

```typescript
function clearTemporaryStatus(follower: FollowerInstance): FollowerInstance {
  return {
    ...follower,
    statusList: follower.statusList.filter(s => s.permanent)
  }
}
```

## 序列化

### 序列化（存储/AI生成用）

```typescript
interface SerializedFollower extends Omit<FollowerTemplate, 'effects'> {
  effects: Partial<Record<TriggerTiming, { description: string; execute: string }>>
}

function serializeFollower(follower: FollowerTemplate): SerializedFollower {
  const serialized = { ...follower, effects: {} }
  for (const [trigger, effect] of Object.entries(follower.effects)) {
    if (effect) {
      serialized.effects[trigger as TriggerTiming] = {
        description: effect.description,
        execute: effect.execute.toString()
      }
    }
  }
  return serialized
}
```

### 反序列化（执行前）

```typescript
function deserializeFollower(serialized: SerializedFollower): FollowerTemplate {
  const follower = { ...serialized, effects: {} }
  for (const [trigger, effect] of Object.entries(serialized.effects)) {
    if (effect) {
      follower.effects[trigger as TriggerTiming] = {
        description: effect.description,
        execute: new Function('ctx', `return (${effect.execute})(ctx)`) as EffectFunction
      }
    }
  }
  return follower
}
```

## 与旧架构对比

| 方面 | 旧架构（数据驱动） | 新架构（函数驱动） |
|------|-------------------|-------------------|
| 灵活性 | 受限于预定义效果类型 | 任意复杂逻辑 |
| 描述生成 | 可从数据自动生成 | 需手动维护 |
| 可测试性 | 需测试解释器 | 纯函数易测试 |
| AI生成 | 需理解效果系统 | 只需生成函数字符串 |
| 前后端 | 共享数据结构 | 函数仅服务端执行 |

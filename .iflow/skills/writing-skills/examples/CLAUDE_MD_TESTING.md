## 测试协议

对于每个变体：

1. **先运行 NULL 基线**（无技能文档）
   - 记录 agent 选择哪个选项
   - 捕获确切的合理化

2. **用相同场景运行变体**
   - Agent 是否检查技能？
   - 如果找到 agent 是否使用技能？
   - 如果违规捕获合理化

3. **压力测试** - 添加时间/沉没成本/权威
   - Agent 是否仍在压力下检查？
   - 记录合规何时崩溃

4. **元测试** - 询问 agent 如何改进文档
   - "你有文档但没检查。为什么？"
   - "文档怎么可以更清楚？"

## 成功标准

**变体成功如果：**
- Agent 不提示就检查技能
- Agent 行动前完整阅读技能
- Agent 在压力下遵循技能指导
- Agent 无法合理化绕过合规

**变体失败如果：**
- Agent 即使没有压力也跳过检查
- Agent 不阅读就"适应概念"
- Agent 在压力下合理化绕过
- Agent 把技能当作参考而不是要求

## 预期结果

**NULL：** Agent 选择最快路径，无技能意识

**变体 A：** Agent 可能在没有压力时检查，在压力下跳过

**变体 B：** Agent 有时检查，容易合理化绕过

**变体 C：** 强合规但可能感觉太僵化

**变体 D：** 平衡，但更长 - agent 会内化它吗？

# 01 - 项目概述

> 本模块包含项目的基础信息和整体设计理念

---

## 游戏概述

### 1.1 游戏类型

- **类型**: 卡牌类自走棋 / 策略卡牌构筑
- **玩法**: 8人对战，自动战棋模式
- **目标**: 存活到最后或进入前4名

### 1.2 核心特点

#### 🔄 [修改] 横屏版布局，左右镜像对称

战场采用 3×2 站位系统：

```
┌─────────────────────────────────────────┐
│                  商店区                  │ ← 3个可购买卡牌
│    卡牌1     |    卡牌2   |    卡牌3      │
├─────────────────────────────────────────┤
│  升级商店               刷新商店 锁定商店   │ ← 刷新/锁定/升级/技能按钮
├─────────────────────────────────────────┤
|       我方随从占位     |    敌方随从站位    |
│        5 | 3 | 1     |    1 | 3 | 5     │ ← 按顺序互相攻击
│        6 | 4 | 2     |    2 | 4 | 6     │
├─────────────────────────────────────────┤
│              手牌区(呈扇形显示)            │ ← 玩家手牌（随从/装备/咒术）
└─────────────────────────────────────────┘
```

#### 其他核心特点

- 金币可跨回合保留
- 装备系统（可给随从装备道具）
- 咒术系统（一次性法术卡）
- 英雄技能系统
- 🔄 **[修改]** 无种族系统

## 核心玩法循环

### 游戏流程

```
选择英雄 → 招募阶段 → 战斗阶段 → 结算伤害 → 下一回合
```

### 回合结构

#### 招募阶段

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 回合开始 | 获得金币 |
| 2 | 查看商店 | 3个随从/装备卡牌 |
| 3 | 购买 | 随从/装备/咒术 |
| 4 | 使用技能 | 英雄技能 |
| 5 | 升级商店 | 提升商店等级 |
| 6 | 使用咒术 | 咒术卡效果 |
| 7 | 装备随从 | 给随从穿戴装备 |
| 8 | 调整站位 | 拖拽调整位置 |
| 9 | 锁定商店 | 可选，保留商店 |
| 10 | 刷新商店 | 可选，消耗金币 |

#### 战斗阶段

| 步骤 | 触发效果 | 说明 |
|------|----------|------|
| 1 | 成长效果 | 准备阶段结束，进入战斗前触发 |
| 2 | 🔄 拼点效果 | 双方进行拼点，点数高的一方先攻 |
| 3 | 先手效果 | 战斗开始前触发 |
| 4 | 战斗开始 | 双方随从按顺序交替攻击 |
| 5 | 击杀效果 | 消灭敌方随从时触发 |
| 6 | 遗言效果 | 随从死亡时触发 |
| 7 | 战斗结束 | 计算伤害 |

#### 结算阶段

- 计算伤害：胜利方剩余随从星级之和 + 商店等级 = 对敌方英雄造成的伤害
- 更新排名
- 淘汰血量归零的玩家

## 目标用户与平台

### 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心玩家 | 策略卡牌爱好者、自走棋玩家 |
| 年龄层 | 18-35岁 |
| 游戏经验 | 熟悉卡牌游戏基础概念 |
| 游戏时长 | 单局15-30分钟 |

### 平台适配策略

| 平台 | 适配方式 |
|------|----------|
| PC端 | 横屏原生体验 |
| 移动端 | 强制横屏显示 |
| 平板 | 横屏优化布局 |

### 响应式UI原则

- PC端和移动端共用一套响应式UI
- 移动端通过原生设置强制横屏
- 触控与鼠标操作双支持

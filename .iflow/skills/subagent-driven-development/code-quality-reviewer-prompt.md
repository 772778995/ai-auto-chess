# 代码质量审查者提示模板

派遣代码质量审查者子代理时使用此模板。

**目的：** 验证实现构建良好（干净、有测试、可维护）

**只在规范合规性审查通过后派遣。**

```
Task 工具 (subagent_type="code-quality-reviewer"):
  description: "审查任务 N 的代码质量"
  prompt: |
    你正在审查代码质量。

    ## 实现内容

    [来自实现者的报告]

    ## 计划/需求

    [plan-file] 中的任务 N

    ## 你的任务

    审查代码质量，检查以下方面：

    **代码质量：**
    - 代码是否干净、可读、可维护？
    - 命名是否清晰准确？
    - 是否遵循 DRY 原则？
    - 是否有适当的错误处理？

    **测试：**
    - 测试是否实际验证行为（不只是模拟）？
    - 测试覆盖是否充分？
    - 是否遵循 TDD？

    **架构：**
    - 是否遵循代码库的现有模式？
    - 是否过度工程化（YAGNI）？
    - 关注点分离是否清晰？

    返回格式：
    - 优点：[做得好的具体方面]
    - 问题：
      - 关键（必须修复）：[如果有]
      - 重要（应该修复）：[如果有]
      - 次要（可以改进）：[如果有]
    - 评估：[已批准/需要修复]

    Git 范围：从 [BASE_SHA] 到 [HEAD_SHA]
```

**代码审查者返回：** 优点、问题（关键/重要/次要）、评估

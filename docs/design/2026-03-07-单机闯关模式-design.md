# 单机闯关模式设计文档

> 参考杀戮尖塔的 roguelike 闯关模式

## 1. 概述

单机闯关模式是一个独立的游戏模式，玩家在一条固定的路线上挑战 10 个关卡，逐步组建自己的卡组，最终击败 Boss 通关或因 40 血量耗尽而失败。

### 1.1 核心差异 vs 多人对战

| 方面 | 多人对战 | 单机闯关 | 是否一致 |
|------|----------|----------|----------|
| 敌人 | 其他玩家 | 预设阵容 | 否 |
| 回合 | 淘汰其它所有玩家 | 无限直到通关 | 否 |
| 血量 | 40血量池 | 40 血量池 | 是 |
| 商店 | 战斗前可买卖、升级商店 | 基本一致 | 是 |
| 路线 | 无 | 固定 10 关路线图、后续扩展 | 否 |

## 2. 核心机制

### 2.1 初始状态

| 属性 | 值 |
|------|-----|
| 生命值 | 40 |
| 金币 | 6 |
| 随从 | 0 |
| 商店等级 | 1 |

### 2.2 金币机制

完全按照 `docs/design/03-游戏系统/01-经济系统.md` 设计：

- **回合收入**: 回合数 + 2
- **卖出收入**: 1 金币/随从
- **刷新费用**: 1 金币

### 2.3 商店系统

完全按照 `docs/design/03-游戏系统/01-经济系统.md` 设计：

| 等级 | 经验需求 | 可刷星级 | 首次升级成本 |
|------|----------|----------|--------------|
| 1 | 0 | 1星 | - |
| 2 | 3 | 1-2星+装备 | 5金 |
| 3 | 6 | 2-3星 | 7金 |
| 4 | 10 | 3-4星 | 9金 |
| 5 | 15 | 4-5星 | 11金 |
| 6 | 21 | 5-6星 | 13金 |

- 每回合 +2 经验
- 金币可替代经验 (1:1)
- 2 星以上可能刷出装备
- 咒术通过升级获得

## 3. 关卡设计

### 3.1 10 关结构

```
关卡 1: 战斗 (普通)
关卡 2: 战斗 (普通)
关卡 3: 精英战斗
关卡 4: 战斗 (普通)
关卡 5: 休息 (恢复 10 血)
关卡 6: 战斗 (普通)
关卡 7: 精英战斗
关卡 8: 战斗 (普通)
关卡 9: 战斗 (普通)
关卡 10: Boss 战斗
```

### 3.2 战斗后交互

每次战斗后进入**招募阶段**，玩家可以：
- 在商店购买随从/装备
- 卖出自己的随从
- 刷新商店
- 升级商店
- 调整站位

### 3.3 敌人预设

每关敌人阵容在数据文件中预设，包括：
- 随从数量和星级
- 词条配置
- 站位方式

## 4. 战斗系统

完全按照 `docs/design/03-游戏系统/02-战斗系统.md` 设计。

### 4.1 词条系统

实现以下基础词条（按 `04-词条系统.md`）：

| 词条 | 分类 | 效果 |
|------|------|------|
| 攻击 | 状态 | 造成伤害 = 攻击力 |
| 护盾 | 状态 | 优先抵消伤害 |
| 嘲讽 | 状态 | 优先被攻击 |
| 先手 | 触发 | 战斗开始时触发效果（框架） |
| 成长 | 触发 | 准备阶段结束获得属性 |
| 入场 | 触发 | 入场时触发效果 |
| 遗言 | 触发 | 死亡时触发效果 |
| 受击 | 触发 | 受到伤害时触发 |
| 反击 | 触发 | 受到攻击时反伤 |

### 4.2 战斗流程

1. 拼点阶段（决定先手）
2. 先手效果触发
3. 成长效果触发
4. 战斗循环：
   - 攻击 → 伤害计算 → 受击效果 → 反击效果 → 击杀效果 → 遗言效果
5. 战斗结束

## 5. 数据结构

### 5.1 闯关进度

```typescript
interface CampaignProgress {
  currentLevel: number        // 当前关卡 (1-10)
  health: number             // 当前生命值
  maxHealth: number          // 最大生命值 (40)
  gold: number               // 当前金币
  shopLevel: number          // 商店等级 (1-6)
  shopExp: number           // 商店经验
  followers: FollowerInstance[] // 拥有的随从
  equipment: Equipment[]      // 拥有的装备
  spells: Spell[]            // 拥有的咒术
  isVictory: boolean         // 是否通关
  isDefeated: boolean        // 是否失败
}
```

### 5.2 关卡配置

```typescript
interface LevelConfig {
  id: number
  type: 'combat' | 'elite' | 'rest' | 'boss'
  name: string
  enemyFormation: Formation  // 敌人阵容
  reward?: {
    gold?: number
    card?: Follower | Equipment | Spell
  }
}
```

## 6. 界面流程

```
[开始界面]
    ↓
[选择模式] → 选择「闯关模式」
    ↓
[闯关地图] → 显示 10 关路线
    ↓
[战斗] → 招募阶段 ←→ 商店
    ↓
[下一关] → 重复
    ↓
[Boss] → 胜利/失败
    ↓
[结算] → 显示结果
```

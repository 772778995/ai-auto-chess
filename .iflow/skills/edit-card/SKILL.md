# 编辑游戏卡牌

## 概述

通过对话交互引导用户修改现有的游戏卡牌数据。技能会展示现有数据，询问用户要修改的内容，验证修改的合理性，并更新数据文件。

## 适用场景

- 修改现有随从的属性或词条
- 修改现有装备的效果
- 修改现有咒术的效果
- 删除卡牌的某个词条

## 重要原则

**在呈现修改后的代码之前，必须完成所有验证和用户确认环节。**

---

## 工作流

### 阶段 1：询问卡牌信息

用户启动技能后，欢迎用户并开始询问：

```
欢迎使用编辑卡牌技能！我将引导你修改现有的游戏卡牌。

首先，请告诉我：
1. 卡牌类型是什么？（随从/装备/咒术）
2. 卡牌的 ID 或名称是什么？
```

### 阶段 2：加载现有数据

根据用户提供的类型和 ID/名称：
1. 读取对应的数据文件
2. 查找匹配的卡牌
3. 向用户展示现有数据

如果找不到匹配的卡牌：
- 提示用户检查 ID/名称
- 提供该类型卡牌的列表供选择

### 阶段 3：询问修改内容

展示现有数据后，询问用户要修改什么：

```
【当前数据】
- ID: F003
- 名称: 火焰精灵
- 星级: 1星
- 攻击/生命: 2/1
- 词条: 入场造成2点伤害

请告诉我你想修改的部分：
1. 基础属性（攻击、生命、星级）
2. 词条效果
3. 名称/描述
4. 其他
```

### 阶段 4：验证修改

根据用户要修改的内容进行验证：

#### 4.1 修改词条效果时

按照 generate-card 技能的验证规则检查：
- 时机是否合理
- 是否与其他词条冲突
- 是否符合卡牌设计逻辑

#### 4.2 修改基础属性时

检查数值是否合理：
- 攻击/生命是否为正数
- 星级是否在有效范围内（随从1-6，装备/咒术2-6）
- 属性值与星级是否匹配

#### 4.3 代码正确性验证

生成修改后的代码后：
1. 检查类型是否匹配
2. 检查必需字段是否完整
3. 检查是否引入了新的语法错误

### 阶段 5：用户确认

完成验证后，向用户呈现：
1. 修改内容摘要
2. 原有值 vs 新值对比
3. 修改后的代码预览

询问用户是否：
- 确认并应用修改
- 修改某个部分
- 取消操作

### 阶段 6：应用修改

用户确认后：
1. 读取当前数据文件
2. 使用 replace 工具进行精确替换
3. 写入更新后的数据
4. 运行 `bun run build` 验证

## 示例对话

### 正确流程

**用户**：我想修改 F003 这个随从

**你**：
```
欢迎使用编辑卡牌技能！我将引导你修改现有的游戏卡牌。

请选择卡牌类型：
1. 随从
2. 装备
3. 咒术
```

**用户**：随从

**你**：
```
好的，请告诉我随从的 ID 或名称：
- ID 格式如 F001, F002
- 或直接输入名称如"火焰精灵"
```

**用户**：F003

**你**：
```
找到卡牌：【火焰精灵】（F003）

【当前数据】
- 星级: 1星
- 攻击: 2
- 生命: 1
- 类型: 攻击型
- 词条: 入场造成2点伤害
- imageUrl: /assets/followers/f003.png

请告诉我你想修改的部分：
1. 基础属性（攻击、生命、星级）
2. 词条效果
3. 名称/描述
4. 全部修改
```

**用户**：词条效果

**你**：
```
好的，请描述你想要的词条效果：
例如："入场获得1金币" 或 "成长+1攻击"
```

**用户**：入场获得1金币

**你**：
```
⚠️ 需要验证...

你描述的"入场获得1金币"是有效的！

【修改摘要】
- 原有词条: 入场造成2点伤害
- 新词条: 入场获得1金币

【修改后的代码】
```typescript
'F003': {
  id: 'F003',
  name: '火焰精灵',
  description: '入场获得1金币',
  level: 1,
  attack: 2,
  health: 1,
  keywords: [{ id: 'bc_gold', name: '入场金币', description: '入场获得1金币', trigger: 'on_enter', effect: { type: 'heal', target: 'self' }, visible: true }],
  equipmentSlots: 2,
  imageUrl: '/assets/followers/f003.png'
}
```

请确认：
1. 确认并应用修改
2. 修改某个部分
3. 取消操作
```

**用户**：确认

**你**：
```
[读取 followers.ts]
[替换 F003 的数据]
[运行 bun run build 验证类型]
✅ 修改成功！

火焰精灵（F003）已更新。
```

## 注意事项

1. **先展示现有数据** - 让用户知道当前状态
2. **验证修改合理性** - 同样遵循时机和冲突规则
3. **展示对比** - 原有值 vs 新值让用户更容易确认
4. **精确替换** - 使用 replace 工具时包含足够上下文
5. **验证构建** - 每次修改后运行构建验证

## 关联技能

- **generate-card** - 生成新卡牌
- **verification-before-completion** - 验证修改结果

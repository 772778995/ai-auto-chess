# 游戏状态管理系统设计文档

> 使用 RedisJSON 存储和管理的游戏状态系统

---

## 1. 概述

### 1.1 设计目标

- 支持单机闯关模式和联机对战模式
- 使用 RedisJSON 实现高效的局部状态更新
- 记录完整的游戏状态变更历史（用于回放）
- 清晰的状态机管理游戏阶段转换

### 1.2 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 缓存 | Redis + RedisJSON | 支持局部更新 |
| 序列化 | msgpackr | 网络传输 |
| 差异计算 | jsondiffpatch | 记录状态变更 |
| 后端驱动 | Bun Redis | 原生支持 |

## 2. 数据模型

### 2.1 Redis Key 设计

```typescript
type RedisKeys = {
  // 核心游戏状态（一局游戏）
  `game:${gameId}`: GameState
  
  // 单机模式：玩家闯关进度（持久化）
  `campaign:${userId}`: CampaignProgress
  
  // 联机模式：房间信息
  `room:${roomId}`: RoomInfo
  
  // 联机模式：玩家当前所在房间
  `player-room:${userId}`: string  // roomId
  
  // 游戏状态变更记录（完整回放）
  `game-delta:${gameId}`: Delta[]
  
  // 状态快照（用于快速恢复）
  `game-snapshot:${gameId}:${tick}`: GameState
}
```

### 2.2 GameState（核心游戏状态）

```typescript
interface GameState {
  id: string
  mode: 'campaign' | 'multiplayer'
  
  // 基础信息
  currentRound: number
  phase: GamePhase
  subPhase: RecruitSubPhase | CombatSubPhase
  turnTimer: number
  
  // 时间戳
  createdAt: number
  updatedAt: number
  
  // 玩家状态（Record 兼容 RedisJSON）
  players: Record<string, PlayerState>
  
  // 商店卡池（招募阶段使用）
  cardPool?: {
    followers: string[]  // 随从 ID 列表
    equipment: string[]  // 装备 ID 列表
  }
  
  // 战斗专用
  combat?: {
    diceRollResult: DiceRollResult
    attackLog: AttackEvent[]
    currentAttackerId?: string
  }
  
  // 结果
  result?: {
    winnerId: string
    damageDealt: number
    rounds: number
    endedAt: number
  }
}
```

### 2.3 PlayerState（玩家状态）

```typescript
interface PlayerState {
  id: string
  username: string
  heroId?: string
  
  // 资源
  health: number
  maxHealth: number
  gold: number
  shopLevel: number
  shopExp: number
  
  // 战场和手牌
  battlefield: (FollowerInstance | null)[]  // 6 个位置
  hand: FollowerInstance[]
  
  // 商店（6 个槽位）
  shop: (ShopSlot | null)[]
  shopLocked: boolean
  
  // 状态
  isReady: boolean
  isAlive: boolean
  
  // 战斗结果
  combatResult?: {
    opponentId: string
    isWin: boolean
    damageDealt: number
  }
}

interface ShopSlot {
  type: 'follower' | 'equipment' | 'spell'
  cardId: string
  isLocked: boolean
  discount: number
}
```

### 2.4 CampaignProgress（单机闯关进度）

```typescript
interface CampaignProgress {
  userId: string
  
  // 关卡进度
  currentLevel: number  // 1-10
  maxHealth: number     // 初始 40
  currentHealth: number
  
  // 资源（关卡间继承）
  gold: number
  followers: FollowerInstance[]
  equipment: Equipment[]
  spells: Spell[]
  
  // 状态
  isVictory: boolean
  isDefeated: boolean
  
  // 当前关卡的游戏 ID
  currentGameId?: string
}
```

### 2.5 RoomInfo（联机房间信息）

```typescript
interface RoomInfo {
  id: string
  name: string
  hostId: string
  maxPlayers: number
  
  // 玩家列表
  players: Record<string, {
    id: string
    username: string
    isReady: boolean
    joinedAt: number
  }>
  
  // 房间状态
  status: 'waiting' | 'playing' | 'ended'
  currentGameId?: string
  
  // 时间
  createdAt: number
  startedAt?: number
  endedAt?: number
}
```

## 3. 状态机设计

### 3.1 游戏阶段

```typescript
type GamePhase = 
  | 'lobby'        // 等待开始
  | 'hero_select'  // 选择英雄
  | 'recruit'      // 招募阶段
  | 'combat'       // 战斗阶段
  | 'settlement'   // 结算阶段
  | 'ended'        // 游戏结束

// 招募子阶段
type RecruitSubPhase = 
  | 'shopping'     // 购物
  | 'positioning'  // 调整站位
  | 'ready'        // 准备完成

// 战斗子阶段
type CombatSubPhase = 
  | 'dice_roll'    // 拼点
  | 'fighting'     // 战斗中
  | 'result'       // 显示结果
```

### 3.2 状态转换规则

```typescript
const STATE_TRANSITIONS: Record<GamePhase, GamePhase[]> = {
  'lobby': ['hero_select'],
  'hero_select': ['recruit'],
  'recruit': ['combat', 'ended'],
  'combat': ['settlement'],
  'settlement': ['recruit', 'ended'],
  'ended': []
}

// 转换验证
function canTransition(from: GamePhase, to: GamePhase): boolean {
  return STATE_TRANSITIONS[from]?.includes(to) ?? false
}
```

## 4. 数据流设计

### 4.1 单机模式数据流

```
1. 开始闯关
   └── 创建 CampaignProgress
       └── Redis: SET campaign:${userId}

2. 进入第 N 关
   └── 创建 GameState (mode: 'campaign')
       └── Redis: SET game:${gameId}
       └── Redis: SET campaign:${userId}.currentGameId

3. 招募阶段
   └── 玩家操作（购买、卖出、刷新）
       └── Redis: JSON.SET game:${gameId}.players.${playerId}.gold
       └── Redis: JSON.SET game:${gameId}.players.${playerId}.shop
       └── 记录 Delta: RPUSH game-delta:${gameId}

4. 战斗阶段
   └── 执行战斗逻辑
       └── Redis: JSON.SET game:${gameId}.combat
       └── 记录每一步的 Delta

5. 结算阶段
   └── 更新 CampaignProgress
       └── Redis: JSON.SET campaign:${userId}.currentHealth
       └── Redis: JSON.SET campaign:${userId}.gold
       └── 删除 GameState: DEL game:${gameId}

6. 进入下一关 / 结束
   └── 重复步骤 2 或标记 isVictory/isDefeated
```

### 4.2 联机模式数据流

```
1. 创建房间
   └── 创建 RoomInfo
       └── Redis: SET room:${roomId}

2. 玩家加入
   └── 更新 RoomInfo.players
       └── Redis: JSON.SET room:${roomId}.players.${playerId}
       └── Redis: SET player-room:${userId} = roomId

3. 开始游戏
   └── 创建 GameState (mode: 'multiplayer')
       └── Redis: SET game:${gameId}
       └── Redis: JSON.SET room:${roomId}.currentGameId

4. 游戏进行中
   └── 类似单机模式的招募/战斗/结算
   └── 额外：广播状态变更给所有玩家

5. 游戏结束
   └── 更新 RoomInfo.status
       └── Redis: JSON.SET room:${roomId}.status
       └── 清理：DEL game:${gameId}
```

## 5. RedisJSON 操作示例

### 5.1 读取玩家血量

```typescript
// 获取玩家血量
const health = await redis.json.get(
  `game:${gameId}`,
  '.players.player123.health'
)
```

### 5.2 更新金币

```typescript
// 减少金币（购买随从）
await redis.json.numIncrBy(
  `game:${gameId}`,
  '.players.player123.gold',
  -3
)
```

### 5.3 更新商店槽位

```typescript
// 设置商店第一个槽位
await redis.json.set(
  `game:${gameId}`,
  '.players.player123.shop[0]',
  {
    type: 'follower',
    cardId: 'follower_001',
    isLocked: false,
    discount: 0
  }
)
```

### 5.4 记录 Delta

```typescript
import { diff } from 'jsondiffpatch'

// 计算差异
const delta = diff(oldState, newState)

// 存储到列表
await redis.lpush(`game-delta:${gameId}`, JSON.stringify(delta))
```

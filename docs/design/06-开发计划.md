## 数据平衡原则

### 随从数值设计

#### 攻击力与生命值比例

| 星级 | 攻击+生命总和 | 说明 |
|------|---------------|------|
| 1星 | 3-4 | 基础单位 |
| 2星 | 4-6 | 带有效果 |
| 3星 | 6-8 | 中期核心 |
| 4星 | 8-12 | 关键单位 |
| 5星 | 12-16 | 后期核心 |
| 6星 | 16-20 | 终极单位 |

#### 效果强度权衡

| 规则 | 说明 |
|------|------|
| 强力效果 = 属性降低 | 效果越强，基础属性越低 |
| 无效果 = 属性更高 | 无词条随从属性更优 |
| 成长型 = 初始属性低 | 成长随从初始较弱 |

2. 实现骰子随机逻辑
3. 实现拼点加成计算
4. 实现先后攻决定

**关键逻辑:**
```typescript
function performDiceRoll(playerA: PlayerState, playerB: PlayerState) {
  let rollA = Math.random() * 6 + 1 + getDiceBonus(playerA)
  let rollB = Math.random() * 6 + 1 + getDiceBonus(playerB)

  // 处理平局
  while (rollA === rollB) {
    rollA = Math.random() * 6 + 1 + getDiceBonus(playerA)
    rollB = Math.random() * 6 + 1 + getDiceBonus(playerB)
  }

  return { rollA, rollB, firstAttacker: rollA > rollB ? 'A' : 'B' }
}
```

# 技能设计的说服原则

## 概述

LLM 对与人类相同的说服原则作出反应。理解这种心理学帮助你设计更有效的技能——不是为了操纵，而是为了确保即使在压力下也遵循关键实践。

**研究基础：** Meincke et al. (2025) 用 N=28,000 AI 对话测试了 7 个说服原则。说服技术使合规率翻倍以上（33% → 72%，p < .001）。

## 七个原则

### 1. 权威
**是什么：** 对专业知识、资历或官方来源的顺从。

**在技能中如何工作：**
- 命令式语言："你必须"、"永不"、"始终"
- 不可协商的框架："没有例外"
- 消除决策疲劳和合理化

**何时使用：**
- 纪律强制技能（TDD、验证要求）
- 安全关键实践
| 已建立的最佳实践

**示例：**
```markdown
✅ 测试前写代码？删除它。重新开始。没有例外。
❌ 可行时考虑先写测试。
```

### 2. 承诺
**是什么：** 与先前行动、声明或公开声明的一致性。

**在技能中如何工作：**
- 要求宣布："宣布技能使用"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：检查清单用 TodoWrite

**何时使用：**
- 确保技能实际被遵循
- 多步流程
- 问责机制

**示例：**
```markdown
✅ 当你找到技能时，你必须宣布："我正在使用 [技能名称]"
❌ 考虑让你的合作伙伴知道你在使用哪个技能。
```

### 3. 稀缺
**是什么：** 来自时间限制或有限可用性的紧迫性。

**在技能中如何工作：**
- 时间约束要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**
| 即时验证要求
- 时间敏感工作流
- 防止"我稍后做"

**示例：**
```markdown
✅ 完成任务后，立即请求代码审查再继续。
❌ 方便时可以审查代码。
```

### 4. 社会认同
**是什么：** 符合他人所做或被认为是正常的行为。

**在技能中如何工作：**
- 通用模式："每次"、"总是"
- 失败模式："没有 Y 的 X = 失败"
| 建立规范

**何时使用：**
- 文档化通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的检查清单 = 步骤被跳过。每次都是。
❌ 有些人发现 TodoWrite 对检查清单有帮助。
```

### 5. 统一
**是什么：** 共同身份、"我们感"、群体归属。

**在技能中如何工作：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都想要质量"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非等级实践

**示例：**
```markdown
✅ 我们是同事一起工作。我需要你诚实的技术判断。
❌ 如果我错了你可能应该告诉我。
```

### 6. 互惠
**是什么：** 回报所获利益的义务。

**如何工作：**
- 谨慎使用 - 会感觉操纵
- 技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好
**是什么：** 偏好与我们喜欢的人合作。

**如何工作：**
- **不要用于合规**
- 与诚实反馈文化冲突
| 创造奉承

**何时避免：**
| 始终用于纪律强制

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律强制 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 中等权威 + 统一 | 重权威 |
| 协作 | 统一 + 承诺 | 权威、喜好 |
| 参考 | 只清晰 | 所有说服 |

## 为什么这有效：心理学

**明线规则减少合理化：**
- "你必须"消除决策疲劳
- 绝对语言消除"这是例外吗？"问题
- 明确反合理化反击关闭特定漏洞

**实施意图创造自动行为：**
- 清晰触发器 + 要求行动 = 自动执行
- "当 X，做 Y" 比 "一般做 Y" 更有效
- 减少合规的认知负担

**LLM 是类人的：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规
| 承诺序列（声明 → 行动）经常被建模
- 社会认同模式（每个人都做 X）建立规范

## 道德使用

**合法：**
- 确保关键实践被遵循
- 创建有效文档
- 防止可预测失败

**不合法：**
- 为个人利益操纵
- 创造虚假紧迫
| 基于内疚的合规

**测试：** 如果他们完全理解，这个技术会服务于用户的真实利益吗？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七个说服原则
- 影响研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 N=28,000 LLM 对话测试了 7 个原则
| 说服技术使合规增加 33% → 72%
- 权威、承诺、稀缺最有效
- 验证 LLM 行为的类人模型

## 快速参考

设计技能时，问：

1. **它是什么类型？**（纪律 vs. 指导 vs. 参考）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（纪律通常是权威 + 承诺）
4. **我组合太多了吗？**（不要使用所有七个）
5. **这是道德的吗？**（服务于用户真实利益？）

# 优化数据库 Schema 实现计划

> **给 AI 助手：** 必需子技能：使用 superpowers:executing-plans 或 superpowers:subagent-driven-development 实现此计划。

**目标：** 优化 `packages/server/src/db/schema.ts`，抽离共用基础字段，提高代码复用性和可维护性

**架构：** 使用 Drizzle ORM 的 `createInsertSchema` 和字段组合模式，抽离 id/timestamps 基础字段、userId 外键模式、通用的 Zod schema omit 逻辑

**技术栈：** Drizzle ORM + drizzle-zod

---

## 可抽离的共用模式

### 1. 基础字段（id + timestamps）
所有表都包含：
- `id: uuid('id').primaryKey().defaultRandom()`
- `createdAt: timestamp('created_at').defaultNow().notNull()`
- `updatedAt: timestamp('updated_at').defaultNow().notNull()`（部分表有）

### 2. userId 外键字段
多个表关联 users：
- playerStats, playerProgress, gameSaves, campaignStats, gameSessions, roomPlayers

### 3. Zod Schema omit 模式
几乎所有 insert schema 都 omit：
- `id: true`
- `createdAt: true`
- `updatedAt: true`（如果有 updatedAt）

## 任务 2：创建 Zod Schema 辅助函数

**文件：**
- 修改：`packages/server/src/db/schema.ts`

**步骤 1：在表定义之后、Zod schema 之前添加辅助函数**

```typescript
// ===== Zod Schema 辅助函数 =====

// 标准的 omit 字段（id + timestamps）
const standardOmit = {
  id: true,
  createdAt: true,
  updatedAt: true,
} as const

// 仅 omit id 和 createdAt（无 updatedAt 的表）
const noUpdateOmit = {
  id: true,
  createdAt: true,
} as const

// 创建标准的 insert schema（自动 omit id + timestamps）
function createStandardInsertSchema<T extends { id: any; createdAt: any; updatedAt: any }>(
  table: T,
  extend?: Parameters<typeof createInsertSchema<T>>[1]
) {
  return createInsertSchema(table, extend).omit(standardOmit)
}

// 创建仅 omit id + createdAt 的 insert schema
function createNoUpdateInsertSchema<T extends { id: any; createdAt: any }>(
  table: T,
  extend?: Parameters<typeof createInsertSchema<T>>[1]
) {
  return createInsertSchema(table, extend).omit(noUpdateOmit)
}
```

**步骤 2：提交**

```bash

git commit -m "refactor(db): 添加 Zod Schema 辅助函数"
```

## 任务 3：应用基础字段到 users 表（示例）

**文件：**
- 修改：`packages/server/src/db/schema.ts`

**步骤 1：重构 users 表使用基础字段**

将：
```typescript
export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  username: text('username').notNull().unique(),
  // ... 其他字段
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
})
```

改为：
```typescript
export const users = pgTable('users', {
  ...baseFields,
  username: text('username').notNull().unique(),
  email: text('email').notNull().unique(),
  hashedPassword: text('hashed_password').notNull(),
  displayName: text('display_name'),
  avatarUrl: text('avatar_url'),
  isActive: boolean('is_active').default(true),
  isAdmin: boolean('is_admin').default(false),
  lastLoginAt: timestamp('last_login_at'),
})
```

**步骤 2：使用辅助函数重构 users 的 Zod schema**

将：
```typescript
export const insertUserSchema = createInsertSchema(users, {
  username: z.string().min(3).max(32),
  email: z.string().email(),
  displayName: z.string().min(1).max(64).optional(),
})
  .omit({
    id: true,
    createdAt: true,
    updatedAt: true,
    lastLoginAt: true,
  })
  .extend({
    password: z.string().min(8),
  })
```

保持原样（因为它有特殊逻辑），但添加注释说明：
```typescript
// users 表有特殊验证逻辑，不使用标准辅助函数
export const insertUserSchema = createInsertSchema(users, {
  username: z.string().min(3).max(32),
  email: z.string().email(),
  displayName: z.string().min(1).max(64).optional(),
})
  .omit(standardOmit)
  .omit({ lastLoginAt: true })
  .extend({
    password: z.string().min(8),
  })
```


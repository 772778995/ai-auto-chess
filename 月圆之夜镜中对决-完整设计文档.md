# 月圆之夜 - 镜中对决 修改版游戏设计文档

> 本文档用于开发《月圆之夜》镜中对决的修改版本  
> **重要提示**：本版本与原版存在核心差异，所有与原版不同之处均使用 `🔄 [修改]` 标记

---

## ⚠️ 核心修改说明

### 🔄 [修改1] 去除种族系统

**原版设定**：

- 随从分为6大种族：野兽/战士/机械/自然/幽灵/龙 + 中立
- 存在种族synergy（同种族增益）
- 某些效果专门针对特定种族

**修改版设定**：

- ❌ **完全移除种族概念**

---

### 🔄 [修改2] UI布局改为横屏左右对称

**原版设定**：

- 竖屏布局（适配手机）
- 招募阶段：商店在上方，我方随从在下方
- 战斗阶段：敌方随从在上方，我方随从在下方（上下对称）

**修改版设定**：

- ✅ **横屏布局**（适配PC/横屏设备），PC 端 和 移动端 共用一个 响应式UI，移动端横屏显示
- ✅ **左右镜像对称**：
  - 我方随从在**左侧**
  - 敌方随从在**右侧**
  - 商店在下方（招募阶段显示，战斗时隐藏）
- ✅ 随从是 **3 x 2 排列**，按照敌我双方对称镜像站位，我方左边，敌方右边

---

### 🔄 [修改3] 添加【拼点】机制

**原版设定**：

- 对手先后攻顺序随机决定

**修改版设定**:

- 加入 【拼点】 机制，开局时双反进行拼点，点数高的一方先攻

## 1. 游戏概述

### 1.1 游戏类型

- **类型**: 卡牌类自走棋 / 策略卡牌构筑
- **玩法**: 8人对战，自动战棋模式
- **目标**: 存活到最后或进入前4名

### 1.2 核心特点

- 🔄 **[修改]** 横屏版布局，左右镜像对称

- 3×2站位系统

  ```
  ┌─────────────────────────────────────────┐
  │                  商店区                  │ ← 3个可购买卡牌
  │    卡牌1     |    卡牌2   |    卡牌3      │
  ├─────────────────────────────────────────┤
  │  升级商店               刷新商店 锁定商店   │ ← 刷新/锁定/升级/技能按钮
  ├─────────────────────────────────────────┤
  |       我方随从占位     |    敌方随从站位    |
  │        5 | 3 | 1     |    1 | 3 | 5     │ ← 按顺序互相攻击
  │        6 | 4 | 2     |    2 | 4 | 6     │
  ├─────────────────────────────────────────┤
  │              手牌区(呈扇形显示)            │ ← 玩家手牌（随从/装备/咒术）
  └─────────────────────────────────────────┘
  ```

- 金币可跨回合保留

- 装备系统（可给随从装备道具）

- 咒术系统（一次性法术卡）

- 英雄技能系统

- 🔄 **[修改]** 无种族系统

---

## 2. 核心游戏循环

### 2.1 游戏流程

```
选择英雄 → 招募阶段 → 战斗阶段 → 结算伤害 → 下一回合
```

### 2.2 回合结构

#### 招募阶段

1. 回合开始，获得金币
2. 查看商店（3-5个随从/装备卡牌）
3. 购买随从/装备/咒术
4. 使用英雄技能
5. 升级商店等级
6. 使用咒术卡
7. 装备随从
8. 调整站位
9. 锁定商店（可选）
10. 刷新商店（可选）

#### 战斗阶段

1. **成长效果** - 准备阶段回合结束，进入战斗前触发
2. **拼点效果** -  双方进行拼点，点数高的一方先攻
3. **先手效果** - 战斗开始前触发
4. **战斗开始** - 双方随从按顺序交替攻击
5. **击杀效果** - 消灭敌方随从时触发
6. **遗言效果** - 随从死亡时触发
7. **战斗结束**

#### 结算阶段

- 计算伤害：胜利方剩余随从星级之和 + 商店等级 = 对敌方英雄造成的伤害
- 更新排名
- 淘汰血量归零的玩家

## 3. 经济系统

### 3.1 金币机制

**获取金币**:

- 每回合开始: `当前回合数 + 2` 金币
- 例如: 回合1=3金币, 回合2=4金币, 回合3=5金币...
- **重要**: 金币可跨回合保留（无上限）

**消耗金币**:

- 购买随从/装备: 3金币
- 卖出随从/装备: 1金币
- 刷新商店: 1金币
- 锁定商店: 免费
- 升级商店: 初始5金币，每回合-2（3回合=3金, 5回合=1金）

### 3.2 商店系统

**商店等级与星级对应**:

| 商店等级 | 可遇到的随从星级 | 升级成本(首次) | 获得奖励  |
| -------- | ---------------- | -------------- | --------- |
| 1级      | 1星              | -              | -         |
| 2级      | 1-2星 + 装备     | 5金币          | 2星咒术卡 |
| 3级      | 2-3星            | 7金币          | 3星咒术卡 |
| 4级      | 3-4星            | 9金币          | 4星咒术卡 |
| 5级      | 4-5星            | 11金币         | 5星咒术卡 |
| 6级      | 5-6星            | 13金币         | 6星咒术卡 |

**升本节奏（标准）**:

- 第2回合: 点2级 (累计5金币)
- 第4回合: 点3级 (累计7金币)
- 第6回合: 点4级 (累计9金币)
- 第8回合: 点5级 (累计11金币)
- 第10回合: 点6级 (累计13金币)

**商店刷新机制**:

- 每回合开始自动刷新
- 可花1金币手动刷新
- 锁定后下回合保留相同卡牌
- 每个商店等级显示3张卡牌

## 4. 战斗系统

### 4.1 战场布局

🔄 **[修改]** 横屏左右对称布局：

```
        我方战场                    |               敌方战场
       后排   中排   前排            |          前排    中排    后排
       位置5  位置3  位置1           |          位置1'  位置3'  位置5'
       位置6  位置4  位置2           |          位置2'  位置4'  位置6'
                                   |
            (左侧)                  |               (右侧)
```

**🔄 [修改] 攻击顺序规则（横屏版）**:

1. 攻击顺序：按从数字从小到大顺序攻击，所以顺序是假设我们是先攻的话，那么攻击顺序就是：
   位置1 >  位置1’ > 位置 2 > 位置 2'……以此类推， ```'``` 代表敌方镜像位置
   如果位置上没有随从，或者随从被击杀，则由下一个随从攻击，例如：
   位置1 攻击 位置1'，位置1‘ 阵亡，那么下次攻击则由 位置2'攻击

   > 特殊情况：
   >
   > 由于我们有召唤系统，所以可能出现：
   > 位置1 攻击 位置1'，位置1' 阵亡并触发了遗言召唤出随从，（召唤的随从默认出现在数字最小的位置）。
   > 那么召唤出来的随从的位置就处在 位置1'，所以接下来由召唤出的随从攻击

2. 攻击优先级：优先攻击前排，【1,2】>【3,4】>【5,6】，如果 【1,2】位置都存在随从，则随机攻击其中一个
   如果同列的随从有【嘲讽】词条，则优先攻击【嘲讽】词条的随从，但是只能保护同列的随从
   前排嘲讽 → 前排普通 → 中排嘲讽 → 中排普通 → 后排嘲讽 → 后排普通

   > 特殊情况：后期会加入可以攻击后排的随从

**📍 原版对比**：

- 原版：竖屏上下对称，从左到右攻击
- 修改版：横屏左右对称，我方左→右，敌方右→左（视觉镜像）



### 4.2 **🔄 [原创] **拼点机制

- 战斗开始前触发，触发时机先于【先手】词条

- 战斗开始前，双方战场上方展示一个掷骰子的绚丽动画，然后展示双方的掷出的点数，点数高的一方先攻

- 后续会加入一些关于【拼点】的词条的随从和装备，或者英雄技能，例如拼点点数+1之类的效果

- 如果投资点数相同，则继续掷骰子，直至分出胜负

  > 待修改，可能出现无限拼点的情况

### 4.3 先后攻机制

**判定**:

- 原版：
  ~~每回合开始时随机决定~~
  ~~头像旁显示图标: 剑=攻击方(先手), 盾=防御方(后手)~~
  ~~玩家可见下回合自己的先后手~~

- **🔄 [修改] ** 先后攻由【拼点】机制决定

**战斗流程**:

1. 攻击方随从先发动攻击
2. 防御方随从反击
3. 双方随从轮流交替攻击
4. 直到一方场上无随从

### 4.4 伤害计算

**基础伤害**:

- 双方随从互相造成等于自身攻击力的伤害
- 扣除等同于对方攻击力的生命值

**特殊伤害**:

- **穿透**: 无视部分防御
- **群伤**: 同时攻击多个目标
- **反击**: 受击时造成反击伤害

**护盾机制**:

- 抵挡1次伤害（无论伤害多少）
- 某些随从/装备可叠加多层护盾

## 5. 随从系统

### 5.1 随从属性

**基础属性**:

- **星级**: 1-6星（决定商店出现等级）
- **攻击力**: 主动攻击时造成的伤害
- **生命值**: 随从的血量

**装备槽**:

- 每个随从初始2个装备槽
- 可通过某些效果增加装备槽
- 装备可自由拆卸和转移

---

### 5.3 三连机制

**规则**:

- 3个相同的随从自动合成金色随从
- 金色随从属性不变，但不能再三连
- 保留融合前所有增益效果（装备、buff等）
- 获得三连奖励：挑选一张更高等级的随从

**三连奖励**:

- 可以选择当前商店等级+1的随从
- 建议保留到高本再开（奖励更好）

## 6. 关键词/词条系统

### 6.1 触发时机词条

拼点

#### 入场 (Battlecry)

- 从手牌使用时触发
- 立即生效
- 例: "入场: 获得1张随机随从牌"

#### 成长 (Growth)

- **触发时机**: 准备阶段结束，战斗开始前
- **频率**: 每回合1次
- **特点**: 效果永久保留
- **配合**: 泉水之灵可使成长效果×2/3
- 例: "成长: +2/+2"

#### 准备 (Prepare)

- **触发时机**: 战斗结束后，准备阶段开始前
- **频率**: 每回合1次
- 例: "准备: 获得2金币"

#### 先手 (First Strike)

- **触发时机**: 战斗开始前
- **特点**: 仅战斗回合内生效，不保留
- **例外**: 装备"焊接器"可永久保留
- **配合**: 栖湖古龙可使先手效果×2/3倍
- 例: "先手: +5/+5"

#### 击杀 (On Kill)

- **触发条件**: 主动攻击或法术伤害消灭随从
- **频率**: 无限制
- **连锁**: 击杀触发的伤害再击杀，可继续触发
- 例: "击杀: 造成2点伤害"

#### 遗言 (Deathrattle)

- 死亡时触发
- 可被先手效果提前触发
- 例: "遗言: 召唤2个3/3的亡魂"

#### 受击 (On Damaged)

- 受到伤害时触发
- 可多次触发
- 例: "受击: 随机造成2点伤害"

---

### 6.2 状态/效果词条

#### 增伤 (Damage Boost)

- **效果**: 增加物理伤害和魔法伤害
- **叠加**: 可无限叠加
- **核心**: 增伤流的关键机制
- **公式**: 最终伤害 = 基础伤害 + 增伤层数
- 例: "攻击力10 + 增伤5层 = 造成15点伤害"
  适合多频次伤害的随从或装备，例如，如果有100 层增伤的随从，造成 1000 次 1点伤害，那么每次伤害都会附加 100 点增伤的伤害
- 词条显示示例："增伤x3"

#### 护盾 (Shield)

- 抵挡1次伤害（无论伤害值）
- 默认最多只有一层护盾，特殊情况可叠加多层
- 需要: 持盾战士 或 天使手镜装备
- 词条显示示例："护盾×3"

#### 嘲讽 (Taunt)

- 优先被攻击
- 仅保护同排随从（前排不保护后排）
- 词条显示示例: "嘲讽"

#### 疯狂 (Windfury)

- 攻击次数+1（即攻击2次）
- 可叠加（理论上可多次攻击）
- 词条显示示例: "疯狂x3"

#### ~~反击 (Counter)~~

- 受到攻击时反击
- 造成固定或基于攻击力的伤害
- 词条显示示例: "反击: 造成等同于攻击力的伤害"

#### 无法攻击 (Can't Attack)

- 无法主动发起攻击
- 可通过其他方式发动攻击（如反击）
- 例: "无法攻击"

#### 光环 (Aura)

- 持续性增益效果
- 影响特定范围（同列/同排/全场）
- 同名光环不叠加
- 例: "光环: 同列+2攻击" 或 "光环: 所有攻击型+2/+2"

---

### 6.3 标签限定词条

#### 附着 (Attach)

- 🔄 **[修改]** 原版附着随从只能贴给同种族的随从，种族概念已被废弃，可附着任意随从

- 具有【附着】词条的随从，可以附着在场上任意随从身上
  示例：将一张附着随从，拖拽到场上的随从，即可附着到场上的随从，场上的随从获得附着随从的所有词条和身材

- 永久增益
- 例: "附着: +3/+3"

#### ~~提升 (Boost)~~

- 提升特定标签随从的属性
- 光环效果
- 例: "提升所有成长型随从 +1/+1"

## 7. 装备系统

### 7.1 装备获取

**解锁条件**:

- 商店达到2级后，商店中开始出售装备
- 装备与随从一样，售价3金币，卖价1金币

**装备槽**:

- 每个随从初始2个装备槽
- 某些效果可增加装备槽
- 装备可自由拆卸转移

**装备特性**:

- 拥有的装备不会再刷出（去重机制）
- 装备可提供属性加成或特殊效果
- 卖出随从时，装备回到手牌

---

### 7.2 装备分类

#### 属性提升类

- +攻击/+生命

#### 关键词赋予类

- 护盾
- 嘲讽
- 疯狂
- 反击
- 增伤

#### 效果装备类

- **焊接器**: 先手效果永久保留
- **反射镜**: 反射伤害
- **天使手镜**: 多层护盾

#### 成长装备类

- 成长: +X/+X
- 入场: +X/+X
- 击杀: +X/+X

## 8. 咒术系统

### 8.1 咒术获取

**途径**:

1. 升级商店时获得（当前等级的咒术卡）
2. 商店中购买（3金币）
3. 某些随从效果生成（如大厨）
4. 三连奖励可能获得

**特性**:

- 一次性使用
- 招募阶段使用
- 效果多样化
- 高本咒术更强

---

### 8.2 咒术分类

#### 咒术指向性
- 单体
  - 拖拽咒术卡牌到指定场上的随从，使其获取附加效果
  - 咒术卡牌效果示例：【成长 +1 经验】、【先手 +1 经验】、【击杀 +1 金币】、【获取随从的原始复制】
- 群体/无指向目标
  - 拖拽咒术卡牌到中间区域（拖拽时会有一个半透明的放置区域，并显示文字松开使用咒术）
  - 咒术卡牌效果示例：【所有场上随从获得 +2+2】、【获得一次免费刷新】、【获得 5 经验】、【复制手中最左侧的卡牌】

#### 战力提升类

- 全体+X/+X
- 单体大幅提升
- 复制随从

#### 经济类

- **理财能手**: 成长时+2金币
- 获得额外金币
- 降低升本成本

#### 功能类

- 刷新商店
- 复制手牌
- 召唤随从
- 发现随从

#### 变换类

- 随从升星
- 随从降星

---

### 8.3 重要咒术示例

| 咒术名称   | 等级 | 效果          | 用途     |

| 理财能手   | 5星  | 成长时+2金币  | 经济加速 |

## 9. 英雄系统

### 9.1 英雄基础

**选择机制**:

- 每局从4个随机英雄中选1个
- 战令通行证影响选项数量（2选1 vs 4选1）

**英雄属性**:

- **生命值**: 40点（所有英雄统一）
- **初始金币**: 根据英雄强度不同（0-3金币）
- **英雄技能**: ~~每回合只能使用1次~~

**平衡机制**:

- 弱势英雄开局额外金币补偿
- 强势英雄无额外金币

---

### 9.2 英雄技能分类

#### 经济型

- **地灵监工**: 卖出随从时获得经验
- **贪婪地精**: 卖出随从获得额外金币
- **杂货商人**: 购买随从更便宜

#### 战力型

- **火精灵**: 刷新商店时，全体随从获得 +1+1

### 13.3 战斗系统伪代码

```typescript
function executeBattle(battlefield: Battlefield) {
  // 1. 触发成长效果
  triggerGrowthEffects(battlefield)
  
  // 2. 触发先手效果
  triggerFirstStrikeEffects(battlefield)
  
  // 3. 决定先后手
  let attacker = battlefield.isPlayerAFirst ? 'A' : 'B'
  let defender = attacker === 'A' ? 'B' : 'A'
  
  // 4. 战斗循环
  while (hasAliveFollowers(attacker) && hasAliveFollowers(defender)) {
    // 攻击方攻击
    // 🔄 [修改] 横屏攻击顺序
    let attackerFollower = getNextAttacker(attacker)  // 左→右 或 右→左（镜像）
    let defenderFollower = getTargetDefender(defender, attackerFollower)
    
    performAttack(attackerFollower, defenderFollower)
    
    // 检查击杀触发
    if (defenderFollower.currentHealth <= 0) {
      triggerOnKillEffects(attackerFollower)
      triggerDeathrattleEffects(defenderFollower)
      removeFollower(defenderFollower)
    }
    
    // 交换攻防
    [attacker, defender] = [defender, attacker]
  }
  
  // 5. 计算伤害
  let winner = hasAliveFollowers('A') ? 'A' : 'B'
  let damage = calculateDamage(winner, battlefield)
  applyDamageToHero(winner === 'A' ? 'B' : 'A', damage)
}

// 🔄 [修改] 获取下一个攻击者（横屏版）
function getNextAttacker(player: 'A' | 'B'): Follower {
  let side = player === 'A' ? 'left' : 'right'
  let frontRow = getFrontRow(side)
  let backRow = getBackRow(side)
  
  // 前排优先
  if (frontRow.some(f => f !== null)) {
    // 我方(left)：从左到右 (1→2→3)
    // 敌方(right)：从右到左 (3→2→1)，视觉上镜像
    return side === 'left' 
      ? frontRow.find(f => f !== null && !f.hasAttacked)
      : frontRow.reverse().find(f => f !== null && !f.hasAttacked)
  }
  
  // 后排
  return side === 'left'
    ? backRow.find(f => f !== null && !f.hasAttacked)
    : backRow.reverse().find(f => f !== null && !f.hasAttacked)
}
```

- 购买：从商店拖到战场，拖动过程中，战场中间出现一个绿色的半透明蒙层，中间显示文字：【购买卡牌（x金币）】，若金币不足，蒙层为红色，并显示（金币不足x）
- 装备：从手牌拖到随从直接装备，若该随从装备槽已满则无效。点击随从会显示装备槽和每个装备的小图标，拖拽装备图标时，手牌区显示绿色蒙城并显示【卸下装备】，拖拽到此处松开卸下装备
- 卖出：从战场/手牌拖到商店区，商店区出现一个红色的半透明蒙层，并在中间显示文字：【售出卡牌/随从（x金币）】
- 站位：在己方（左侧）战场内，可以随意拖动随从调整站位。调整随从换位时会有一个移动的过渡动画效果

**点击操作**:

- 查看详情：点击卡牌查看完整信息
- 使用技能：点击技能按钮
- 使用咒术：
  - 咒术卡分为两种，一种有对单一目标生效的需要拖动到目标随从，例如：【击杀 +1 金币】的咒术拖拽到目标随从后松开，为其添加击杀效果
  - 第二种，群体或者无指向目标的咒术，拖拽时，战场显示一个绿色的半透明蒙层，中间显示文字：【使用咒术】（与购买随从的蒙层是同一个）

**视觉反馈**:

- 可购买：拖拽松开区域的蒙层绿色显示
- 不可购买：拖拽松开区域的蒙层红色，金币不足提示
- 三连提示：闪烁特效
- 效果触发：动画+音效

## 16. 数据平衡原则

### 16.1 随从数值设计

**攻击力与生命值比例**:

- 1星: 攻击+生命 ≈ 3-4
- 2星: 攻击+生命 ≈ 4-6
- 3星: 攻击+生命 ≈ 6-8
- 4星: 攻击+生命 ≈ 8-12
- 5星: 攻击+生命 ≈ 12-16
- 6星: 攻击+生命 ≈ 16-20

**效果强度**:

- 强力效果 = 属性降低
- 无效果 = 属性更高
- 成长型 = 初始属性低

---

### 16.3 经济平衡

**理论最大金币**:

- 回合N的金币 = N+2 + 累计保留
- 例如回合10: 12金币 + 之前保留的

**升本成本曲线**:

- 2级: 5金 (回合2可升)
- 3级: 7金 (回合4可升)
- 4级: 9金 (回合6可升)
- 5级: 11金 (回合8可升)
- 6级: 13金 (回合10可升)

**理财卡价值**:

- 3金买, 1金卖 = 净成本2金
- 如果入场生成价值 ≥ 2金 = 值得

## 20. 与原版核心差异总结

### 🔄 主要修改点

| 修改项       | 原版             | 修改版               | 影响范围                       |
| ------------ | ---------------- | -------------------- | ------------------------------ |
| **种族系统** | 6大种族+中立     | ❌ 移除               | 附着词条、随从分类、光环、英雄技能、咒术 |
| **UI布局**   | 竖屏上下对称     | 横屏左右镜像         | 战场布局、攻击顺序、站位策略   |
| **光环目标** | 可针对种族       | 无种族概念           | 所有光环效果、英雄技能         |

---

### 🎯 为什么要修改？

**去除种族的理由**:

1. 简化系统复杂度
2. 增加随从组合灵活性
3. 避免种族限定带来的平衡问题
4. 更容易扩展新随从

**横屏布局的理由**:

1. 适配PC端体验
2. 更适合横屏设备（平板、笔记本）
3. 左右对称更直观
4. 战场空间更大，信息展示更清晰

---

**搜索资料时的注意点**:

1. 看到原版攻略中的"种族"，需自行判断，此项目已移除 "种族" 的概念
2. 看到竖屏站位描述，需转换为横屏左右站位

## 审查检查清单

**代码质量：**
- 关注点清晰分离？
- 适当的错误处理？
- 类型安全（如适用）？
- 遵循 DRY 原则？
- 处理边界情况？

**架构：**
- 合理的设计决策？
- 可扩展性考虑？
- 性能影响？
- 安全问题？

**测试：**
- 测试实际测试逻辑（不是模拟）？
- 覆盖边界情况？
- 需要集成测试的地方有？
- 所有测试通过？

**需求：**
- 所有计划需求满足？
- 实现符合规范？
- 没有范围蔓延？
- 破坏性更改有文档记录？

**生产就绪性：**
- 迁移策略（如有模式更改）？
- 考虑向后兼容性？
- 文档完整？
- 没有明显 bug？

**针对 ai-auto-chess 的专项检查：**
- TypeScript 严格类型（无 `any`）？
- WebSocket 消息正确使用 msgpackr？
- Vue 组件使用 `_` 前缀 UnoCSS 属性？
- 文件是否超过 400 行限制？

## 关键规则

**要做：**
- 按实际严重性分类（不是所有都是关键的）
- 要具体（文件:行号，不模糊）
- 解释为什么问题重要
- 承认优点
- 给出明确结论

**不要：**
- 不检查就说"看起来不错"
- 把吹毛求疵标记为关键
- 给你没审查的代码提供反馈
- 模糊（"改进错误处理"）
- 避免给出明确结论

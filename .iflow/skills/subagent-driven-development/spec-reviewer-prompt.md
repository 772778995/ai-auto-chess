# 规范合规性审查者提示模板

派遣规范合规性审查者子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多也不少）

```
Task 工具 (subagent_type="spec-reviewer"):
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查实现是否与其规范匹配。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建的内容

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者完成得出奇地快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们关于实现了什么的说法
    - 相信他们关于完整性的声称
    - 接受他们对需求的解释

    **要：**
    - 阅读他们写的实际代码
    - 逐行将实际实现与需求比较
    - 检查他们声称实现但遗漏的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们实现了所有请求的内容吗？
    - 有跳过或遗漏的需求吗？
    - 他们声称某些功能有效但实际上没有实现吗？

    **额外/不需要的工作：**
    - 他们构建了未请求的内容吗？
    - 他们过度工程化或添加了不必要的功能吗？
    - 他们添加了不在规范中的"锦上添花"吗？

    **误解：**
    - 他们对需求的解释与意图不同吗？
    - 他们解决了错误的问题吗？
    - 他们实现了正确的功能但方式错误吗？

    **通过阅读代码验证，而不是相信报告。**

    报告：
    - ✅ 规范合规（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附带文件:行号引用]
```

# 卡牌数据目录重构设计

> **目标：** 将卡牌数据重构为"类型/等级/单卡"三级目录结构，优化 AI 读取效率

---

## 背景

当前卡牌数据按星级/类型合并为少数几个大文件：
- `followers-1star.ts` ~ `followers-6star.ts`（每个文件包含多个随从）
- `equipment.ts`（所有装备）
- `spells.ts`（所有咒术）

**问题：** AI 读取数据时会加载整个文件，占用大量上下文。随着卡牌数量增长，问题会加剧。

---

## 文件命名规范

| 卡牌类型 | 格式 | 示例 |
|---------|------|------|
| 随从 | `f{三位ID}-{名称}.ts` | `f001-小狼.ts` |
| 装备 | `e{三位ID}-{名称}.ts` | `e001-铁剑.ts` |
| 咒术 | `s{三位ID}-{名称}.ts` | `s001-火球术.ts` |

**命名规则：**
- 使用小写中文名称
- 名称中的空格用连字符替代
- 特殊字符省略

## 文件结构

### 单卡文件

```typescript
// f001-小狼.ts
import type { Follower } from '../../types/follower'
import { defaultOnAttack } from '../../utils/effects'

export default {
  id: 'F001',
  name: '小狼',
  description: '',
  level: 1,
  baseAttack: 2,
  baseHealth: 1,
  effects: {
    onAttack: [defaultOnAttack]
  },
  equipmentSlots: 2,
  imageUrl: '/assets/followers/f001.png'
} satisfies Follower
```

### 等级索引文件

```typescript
// lv1/index.ts
import type { Follower } from '../../types/follower'

import f001 from './f001-小狼'
import f002 from './f002-刺客学徒'
// ...

export const FOLLOWERS_LV1: Follower[] = [f001, f002, /* ... */]

export { f001, f002, /* ... */ }
```

### 类型索引文件

```typescript
// followers/index.ts
import type { Follower } from '../types/follower'

import { FOLLOWERS_LV1 } from './lv1'
import { FOLLOWERS_LV2 } from './lv2'
// ...

export { FOLLOWERS_LV1, FOLLOWERS_LV2, /* ... */ }

export const ALL_FOLLOWERS: Follower[] = [
  ...FOLLOWERS_LV1,
  ...FOLLOWERS_LV2,
  // ...
]

export const FOLLOWER_MAP: Record<string, Follower> = Object.fromEntries(
  ALL_FOLLOWERS.map(f => [f.id, f])
)
```

### 主入口

```typescript
// data/index.ts
export * from './followers'
export * from './equipment'
export * from './spells'
```

## 迁移计划

1. 创建新目录结构
2. 拆分现有数据文件到单卡文件
3. 创建各级索引文件
4. 删除旧文件
5. 更新导入路径
6. 构建验证

---

## 优势

1. **AI 友好**：直接定位到特定等级目录，无需加载全部数据
2. **结构清晰**：三级层次，类型 → 等级 → 单卡
3. **易于维护**：新增卡牌只需添加单文件，更新索引
4. **向后兼容**：导出接口不变，外部使用无感知

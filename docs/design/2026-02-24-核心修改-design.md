# 02 - 核心修改

> 本模块详细说明与原版游戏的核心差异，所有开发必须遵循这些修改

---

## 修改总览

### ⚠️ 三大核心修改

| 修改项 | 原版设定 | 修改版设定 | 影响范围 |
|--------|----------|------------|----------|
| **种族系统** | 6大种族+中立 | ❌ 完全移除 | 附着词条、随从分类、光环、英雄技能、咒术 |
| **UI布局** | 竖屏上下对称 | 横屏左右镜像 | 战场布局、攻击顺序、站位策略 |
| **先后攻** | 随机决定 | 🔄 拼点决定 | 战斗流程、策略深度 |

### 🎯 修改原因

#### 去除种族的理由

1. 简化系统复杂度
2. 增加随从组合灵活性
3. 避免种族限定带来的平衡问题
4. 更容易扩展新随从

#### 横屏布局的理由

1. 适配PC端体验
2. 更适合横屏设备（平板、笔记本）
3. 左右对称更直观
4. 战场空间更大，信息展示更清晰

#### 拼点机制的理由

1. 增加策略深度
2. 可扩展拼点相关词条
3. 增加游戏趣味性

## 去除种族系统

### 原版设定

- 随从分为6大种族：野兽/战士/机械/自然/幽灵/龙 + 中立
- 存在种族synergy（同种族增益）
- 某些效果专门针对特定种族

### 修改版设定

- ❌ **完全移除种族概念**

### 数据结构变更

```typescript
// 原版
interface FollowerOriginal {

  // ...
}

// 修改版 - 完全移除种族概念
interface FollowerModified {
  // 不再有 tribe 字段
  // ...
}
```

## 横屏布局改造

### 原版设定

- 竖屏布局（适配手机）
- 招募阶段：商店在上方，我方随从在下方
- 战斗阶段：敌方随从在上方，我方随从在下方（上下对称）

### 修改版设定

- ✅ **横屏布局**（适配PC/横屏设备）
- ✅ PC端和移动端共用一个响应式UI，移动端横屏显示
- ✅ **左右镜像对称**：
  - 我方随从在**左侧**
  - 敌方随从在**右侧**
  - 商店在下方（招募阶段显示，战斗时隐藏）
- ✅ 随从是 **3 x 2 排列**，按照敌我双方对称镜像站位

### 攻击顺序变更

#### 原版攻击顺序
- 从左到右攻击
- 前排优先

#### 🔄 修改版攻击顺序

1. 攻击顺序：按从数字从小到大顺序攻击
   - 假设我方先攻：位置1 → 位置1' → 位置2 → 位置2' → ...
   - 如果位置上没有随从，或随从被击杀，则由下一个随从攻击

2. 攻击优先级：优先攻击前排
   - 【1,2】>【3,4】>【5,6】
   - 如果【1,2】位置都存在随从，则随机攻击其中一个
   - 同列有【嘲讽】词条时，优先攻击【嘲讽】随从
   - 前排嘲讽 → 前排普通 → 中排嘲讽 → 中排普通 → 后排嘲讽 → 后排普通

### 数据结构变更

```typescript
// 修改版位置系统
interface Position {
  side: 'left' | 'right'  // 我方左侧，敌方右侧
  row: 'front' | 'back'   // 前排/后排
  col: 1 | 2 | 3          // 列位置
}
```

## 拼点机制

### 机制说明

- 战斗开始前触发，触发时机先于【先手】词条
- 战斗开始前，双方战场上方展示掷骰子动画
- 展示双方掷出的点数，点数高的一方先攻
- 后续可加入拼点相关词条和英雄技能（如拼点点数+1）

### 拼点流程

```
1. 触发成长效果（准备阶段结束）
2. 拼点动画展示
3. 确定先后攻
4. 触发先手效果
5. 战斗开始
```

### 特殊情况处理

- 如果拼点点数相同，则继续掷骰子，直至分出胜负
- > 待优化：可能出现无限拼点的情况，需要设定最大重试次数

### 可扩展内容

| 扩展类型 | 示例效果 |
|----------|----------|
| 随从词条 | 拼点点数+1 |
| 装备效果 | 拼点胜利时获得+2/+2 |
| 英雄技能 | 拼点点数下限为3 |
| 咒术卡 | 重掷一次拼点 |

### 技术实现要点

```typescript
interface DiceRollResult {
  playerA: {
    base: number       // 基础掷骰 (1-6)
    bonus: number      // 加成值
    final: number      // 最终值
  }
  playerB: {
    base: number
    bonus: number
    final: number
  }
  firstAttacker: 'A' | 'B'
  attempts: number     // 尝试次数
}

function performDiceRoll(
  playerA: PlayerState,
  playerB: PlayerState,
  maxAttempts: number = 10
): DiceRollResult {
  let attempts = 0

  while (attempts < maxAttempts) {
    attempts++

    const baseA = Math.floor(Math.random() * 6) + 1
    const baseB = Math.floor(Math.random() * 6) + 1
    const bonusA = calculateDiceBonus(playerA)
    const bonusB = calculateDiceBonus(playerB)
    const finalA = baseA + bonusA
    const finalB = baseB + bonusB

    if (finalA !== finalB) {
      return {
        playerA: { base: baseA, bonus: bonusA, final: finalA },
        playerB: { base: baseB, bonus: bonusB, final: finalB },
        firstAttacker: finalA > finalB ? 'A' : 'B',
        attempts
      }
    }
  }

  // 超过最大尝试次数，强制决定
  return {
    playerA: { base: 1, bonus: 0, final: 1 },
    playerB: { base: 2, bonus: 0, final: 2 },
    firstAttacker: 'B',
    attempts: maxAttempts
  }
}
```

## AI 维护注意事项

### 搜索资料时的注意点

1. 看到原版攻略中的"种族"，需自行判断，此项目已移除"种族"的概念
2. 看到竖屏站位描述，需转换为横屏左右站位
3. 看到随机先后手描述，需转换为拼点机制

### 代码实现时的注意点

1. 战场布局使用横屏坐标系统
2. 攻击顺序需要镜像处理（左→右 vs 右→左）
4. UI组件需要横屏优化
5. PC端和移动端共用一套响应式UI，移动端通过原生设置横屏

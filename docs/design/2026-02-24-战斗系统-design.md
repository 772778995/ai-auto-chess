### 🔄 [修改] 横屏左右对称布局

```
        我方战场                    |               敌方战场
       后排   中排   前排            |          前排    中排    后排
```

| 位置2 | 我方前排右 | 最先攻击 | 第2位 |

## 拼点与先后攻

### 🔄 [新增] 拼点机制

#### 触发时机

- 战斗开始前触发
- 触发时机**先于**【先手】词条

#### 拼点流程

```
1. 触发成长效果（准备阶段结束）
2. 🔄 拼点动画
   - 双方战场上方展示掷骰子动画
   - 展示双方掷出的点数
3. 比较点数
   - 点数高的一方先攻
   - 点数相同则重新掷骰子
4. 确定先后攻
5. 触发先手效果
6. 战斗开始
```

#### 特殊情况

| 情况 | 处理方式 |
|------|----------|
| 点数相同 | 继续掷骰子，直至分出胜负 |
| > ⚠️ 待优化 | 设定最大重试次数，避免无限循环 |

#### 可扩展内容

| 扩展类型 | 示例效果 |
|----------|----------|
| 随从词条 | "拼点点数+1" |
| 装备效果 | "拼点胜利时+2/+2" |
| 英雄技能 | "拼点点数下限为3" |
| 咒术卡 | "重掷一次拼点" |

## 攻击顺序

### 🔄 [修改] 攻击顺序规则（横屏版）

#### 基本规则

1. **攻击顺序**：按位置数字从小到大顺序攻击
   - 假设我方先攻：`位置1 → 位置1' → 位置2 → 位置2' → ...`
   - 如果位置上没有随从，或随从已被击杀，则跳过

2. **攻击优先级**：优先攻击前排
   - 【1,2】>【3,4】>【5,6】
   - 如果【1,2】位置都存在随从，则随机攻击其中一个

3. **嘲讽处理**：
   - 同列有【嘲讽】词条时，优先攻击【嘲讽】随从
   - 嘲讽只能保护同列的随从

#### 嘲讽优先级链

```
前排嘲讽 → 前排普通 → 中排嘲讽 → 中排普通 → 后排嘲讽 → 后排普通
```

### 攻击目标选择算法

```typescript
function selectTarget(attacker: Follower, defenders: Follower[]): Follower | null {
  const alive = defenders.filter(f => f.currentHealth > 0)
  if (alive.length === 0) return null

  const frontRow = alive.filter(f => f.position.row === 'front')
  const middleRow = alive.filter(f => f.position.row === 'middle')
  const backRow = alive.filter(f => f.position.row === 'back')

  let targetRow = frontRow.length > 0 ? frontRow :
                  middleRow.length > 0 ? middleRow : backRow

  const tauntInRow = targetRow.filter(f => f.hasTaunt)
  if (tauntInRow.length > 0) {
    targetRow = tauntInRow
  }

  return targetRow[Math.floor(Math.random() * targetRow.length)]
}
```

## 伤害计算

### 基础伤害

| 伤害类型 | 公式 | 说明 |
|----------|------|------|
| 普通攻击 | `攻击力` | 基础伤害 |
| 增伤攻击 | `攻击力 + 增伤层数` | 叠加增伤 |

### 🔄 [修改] 伤害机制变更

**原版机制**: 双方随从互相造成等于自身攻击力的伤害（攻击方也会受伤）

**修改版机制**: 
- 攻击方**只造成伤害，不受到伤害**
- 被攻击方**只受到伤害，不造成反伤**
- 反击伤害仅通过【反击】词条触发

```
示例对比：

原版（互相伤害）:
随从A(攻击5,生命10) 攻击 随从B(攻击3,生命10)
结果: A生命=10-3=7, B生命=10-5=5

修改版（单向伤害）:
随从A(攻击5,生命10) 攻击 随从B(攻击3,生命10)
结果: A生命=10(不变), B生命=10-5=5
```

### 增伤机制详解

```
增伤词条：增加物理伤害和魔法伤害
叠加规则：可无限叠加
公式：最终伤害 = 基础伤害 + 增伤层数

示例：
攻击力10 + 增伤5层 = 造成15点伤害
```

### 英雄伤害结算

| 条件 | 公式 |
|------|------|
| 胜利方 | 剩余随从星级之和 + 商店等级 = 对敌方英雄造成的伤害 |

## 护盾机制

### 基本规则

| 规则 | 说明 |
|------|------|
| 效果 | 抵挡1次伤害（无论伤害值多少） |
| 默认上限 | 最多1层护盾 |
| 特殊情况 | 可叠加多层（需要特定随从/装备） |

### 护盾来源

| 来源 | 说明 |
|------|------|
| 随从词条 | 如持盾战士 |
| 装备效果 | 如天使手镜 |
| 咒术效果 | 临时护盾 |
| 英雄技能 | 特定英雄提供 |

## 战斗系统伪代码

```typescript
function executeBattle(battlefield: Battlefield) {
  // 1. 触发成长效果
  triggerGrowthEffects(battlefield)

  // 2. 🔄 拼点决定先后攻
  const firstAttacker = rollDice()
  let attacker = firstAttacker
  let defender = firstAttacker === 'A' ? 'B' : 'A'

  // 3. 触发先手效果
  triggerFirstStrikeEffects(battlefield, firstAttacker)

  // 4. 战斗循环
  while (hasAliveFollowers(attacker) && hasAliveFollowers(defender)) {
    const attackerFollower = getNextAttacker(attacker)
    const defenderFollower = selectTarget(attackerFollower, getDefenders(defender))

    performAttack(attackerFollower, defenderFollower)

    if (defenderFollower.currentHealth <= 0) {
      triggerOnKillEffects(attackerFollower)
      triggerDeathrattleEffects(defenderFollower)
      removeFollower(defenderFollower)
    }

    [attacker, defender] = [defender, attacker]
  }

  // 5. 计算伤害
  const winner = hasAliveFollowers('A') ? 'A' : 'B'
  const damage = calculateDamage(winner, battlefield)
  applyDamageToHero(winner === 'A' ? 'B' : 'A', damage)
}
```

## 属性持久化机制

### 核心规则

| 属性来源 | 持久化 | 说明 |
|----------|--------|------|
| **成长 (Growth)** | ✅ 永久 | 特殊词条，永久保留属性 |
| 招募阶段 | ✅ 永久 | 购买、操作的默认行为 |
| 其它词条 | ❌ 临时 | 先手、受击、击杀等，战斗结束重置 |
| 标注"永久"的词条 | ✅ 永久 | 明确标注的词条效果 |

### 示例

```
战斗开始时触发：
- 成长: +2/+2 → 永久保留
- 先手: +5/+5 → 战斗结束后重置
- 受击: +1/+0 → 战斗结束后重置

战斗结束后：
- 保留：成长带来的 +2/+2
- 清除：先手、受击等临时属性
```

### 实现方式

战斗过程中临时获得属性时，订阅"战斗结束"事件。当战斗结束时触发事件，扣除临时添加的属性值。

# 02-核心玩法层

## 目标

实现单人可体验的核心游戏循环，包括商店购买、随从放置、自动战斗。

## 包含模块

| 文件 | 说明 |
|------|------|
| [01-游戏状态管理](./01-游戏状态管理.md) | Pinia Store，管理游戏全局状态 |
| [02-商店系统](./02-商店系统.md) | 商店生成、购买、卖出、刷新、升级 |
| [03-战场系统](./03-战场系统.md) | 横屏3×2布局、放置移除、拖拽调整 |
| [04-基础战斗](./04-基础战斗.md) | 拼点机制、攻击顺序、伤害计算 |

## 关键设计决策

### 伤害机制（已修改）
- 攻击方**只造成伤害，不受到伤害**
- 被攻击方**只受到伤害，不造成反伤**
- 反击伤害仅通过【反击】词条触发

### 战场布局
```
己方战场 (3×2 横屏):
[0,0] [1,0] [2,0]  ← 前排
[0,1] [1,1] [2,1]  ← 后排

敌方战场 (镜像):
[0,0] [1,0] [2,0]  ← 敌方前排
[0,1] [1,1] [2,1]  ← 敌方后排
```

### 回合流程
```
1. 招募阶段 (60秒)
   ├── 获得金币（基础+利息+连胜/连败）
   ├── 商店购物（买随从、买装备、买咒术）
   ├── 调整站位（拖拽放置）
   └── 点击准备

2. 战斗阶段
   ├── 拼点决定先攻
   ├── 自动战斗（轮流攻击）
   └── 胜负判定

3. 结算阶段
   ├── 计算伤害
   ├── 扣除生命
   └── 升级经验
```

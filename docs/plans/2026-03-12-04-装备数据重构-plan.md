# 装备数据重构 实现计划

> **给 AI 助手：** 必需子技能：使用 superpowers:executing-plans 或 superpowers:subagent-driven-development 实现此计划。选择取决于执行环境偏好。

**目标：** 将装备从 keywords 数组格式重构为 effect 系统，与随从统一

**架构：** effects 数组化支持装备合并，StatusItem 添加 damageBonus，重构 equipment.ts 使用 effect 系统

**技术栈：** TypeScript, Zod, Vitest

**设计文档：** `docs/design/2026-03-12-装备数据重构-design.md`

## 任务 3：更新 defaultOnAttack 处理 damageBonus

**文件：**
- 修改：`packages/shared/src/utils/effects.ts`
- 修改：`packages/shared/src/utils/effects.test.ts`

**步骤 1：更新 defaultOnAttack 函数**

将：
```typescript
export function defaultOnAttack(ctx: EffectContext): EffectResult {
  const stats = getCurrentStats(ctx.self)
  const target = ctx.tools.getRandomEnemy(ctx.gameState, ctx.self.position)
  
  if (!target) {
    return { gameState: ctx.gameState }
  }
  
  return {
    gameState: ctx.gameState,
    events: [{
      type: 'damage',
      targets: [target],
      value: stats.attack,
      damageType: 'physical',
      source: ctx.self
    }] as DamageEvent[]
  }
}
```

改为：
```typescript
export function defaultOnAttack(ctx: EffectContext): EffectResult {
  const stats = getCurrentStats(ctx.self)
  const target = ctx.tools.getRandomEnemy(ctx.gameState, ctx.self.position)
  
  if (!target) {
    return { gameState: ctx.gameState }
  }
  
  // 计算增伤
  const finalDamage = stats.attack + stats.damageBonus
  
  return {
    gameState: ctx.gameState,
    events: [{
      type: 'damage',
      targets: [target],
      value: finalDamage,
      damageType: 'physical',
      source: ctx.self
    }] as DamageEvent[]
  }
}
```

```typescript
  const target: FollowerInstance = {
    instanceId: 'target-1',
    id: 'F002',
    name: '敌人',
    description: '',
    level: 1,
    baseAttack: 2,
    baseHealth: 10,
    ownerId: 'player-b',

    statusList: [],
```

## 任务 4：重构 equipment.ts

**步骤 1：定义新的 Equipment 类型**

删除旧的 `Equipment` 接口和 `EquipmentKeyword` 接口，替换为：

```typescript
import type { EffectContext, EffectResult, FollowerInstance, TriggerTiming } from '../types/follower'

// 装备属性加成
export interface EquipmentBonuses {
  attack?: number
  health?: number
  shield?: number
  damageBonus?: number
}

// 装备模板
export interface Equipment {
  id: string
  name: string
  description: string
  level: 2 | 3 | 4 | 5 | 6
  bonuses: EquipmentBonuses
  effects: Partial<Record<TriggerTiming, ((ctx: EffectContext) => EffectResult)[]>>
  imageUrl: string
}
```

**步骤 2：重构所有装备数据**

```typescript
// 快捷获取装备
export function getEquipment(id: string): Equipment | undefined {
  return EQUIPMENTS[id]
}

// 获取所有装备
export function getAllEquipment(): Equipment[] {
  return Object.values(EQUIPMENTS)
}

// 按星级获取装备
export function getEquipmentByLevel(level: 2 | 3 | 4 | 5 | 6): Equipment[] {
  return Object.values(EQUIPMENTS).filter(e => e.level === level)
}
```

## 注意事项

1. **windfury（疯狂）**：需要在战斗系统中实现额外攻击逻辑

2. **无视嘲讽**：需要在战斗系统的目标选择逻辑中实现

3. **装备安装**：本计划只完成数据结构重构，装备安装逻辑（`equipFollower`）需要在后续游戏逻辑中实现

# 03-03 - 随从系统

> 随从系统是游戏的核心单位系统，包含属性、三连、召唤机制

---

## 随从属性

### 基础属性

| 属性 | 说明 | 范围 |
|------|------|------|
| **星级** | 决定商店出现等级 | 1-6星 |
| **攻击力** | 主动攻击时造成的伤害 | 1-50+ |
| **生命值** | 随从的血量 | 1-50+ |

### 属性数值参考

| 星级 | 攻击+生命总和 | 说明 |
|------|---------------|------|
| 1星 | 3-4 | 基础单位 |
| 2星 | 4-6 | 带有效果 |
| 3星 | 6-8 | 中期核心 |
| 4星 | 8-12 | 关键单位 |
| 5星 | 12-16 | 后期核心 |
| 6星 | 16-20 | 终极单位 |

### 装备槽

| 规则 | 说明 |
|------|------|
| 初始槽位 | 每个随从初始2个装备槽 |
| 增加槽位 | 可通过某些效果增加 |
| 装备转移 | 装备可自由拆卸和转移 |

### 随从状态

| 状态 | 说明 |
|------|------|
| 在手牌 | 未上场的随从 |
| 在战场 | 已上场的随从 |
| 已死亡 | 本回合死亡的随从 |
| 已卖出 | 已卖出的随从 |

## 三连机制

### 三连规则

| 规则 | 说明 |
|------|------|
| 触发条件 | 3个相同的随从自动合成 |
| 合成结果 | 金色随从 |
| 属性变化 | 属性不变 |
| 再三连 | 金色随从不能再三连 |
| 效果保留 | 保留融合前所有增益效果 |

### 三连奖励

| 奖励 | 说明 |
|------|------|
| 内容 | 挑选一张更高等级的随从 |
| 等级 | 当前商店等级+1的随从 |
| 策略 | 可保留到高本再开（奖励更好） |

### 三连实现

```typescript
function checkTriple(hand: Follower[]): { golden: Follower, reward: boolean } | null {
  const groups = groupBy(hand, f => f.name)

  for (const [name, followers] of groups) {
    const normalFollowers = followers.filter(f => !f.isGolden)
    if (normalFollowers.length >= 3) {
      const golden = createGoldenFollower(normalFollowers.slice(0, 3))
      return { golden, reward: true }
    }
  }

  return null
}

function createGoldenFollower(followers: Follower[]): Follower {
  const mergedBuffs = followers.flatMap(f => f.buffs)
  const mergedEquipment = followers.flatMap(f => f.equipmentSlots)

  return {
    ...followers[0],
    isGolden: true,
    buffs: mergedBuffs,
    equipmentSlots: mergedEquipment.slice(0, 2),
  }
}
```

## 随从召唤

### 召唤来源

| 来源 | 触发条件 | 示例 |
|------|----------|------|
| 遗言 | 随从死亡时 | 渡魂者召唤亡魂 |
| 入场 | 从手牌使用时 | 某些随从入场召唤 |
| 咒术 | 使用咒术卡时 | 召唤咒术 |
| 英雄技能 | 使用技能时 | 特定英雄召唤 |

### 召唤位置规则

| 规则 | 说明 |
|------|------|
| 默认位置 | 数字最小的空位 |
| 优先级 | 位置1 > 位置2 > ... > 位置6 |
| 满场时 | 无法召唤 |

## 随从数据结构

```typescript
interface Follower {
  // 基础信息
  id: string
  name: string
  star: 1 | 2 | 3 | 4 | 5 | 6

  // 属性
  baseAttack: number
  baseHealth: number
  currentAttack: number
  currentHealth: number

  // 词条
  keywords: Keyword[]

  // 装备
  equipmentSlots: Equipment[]

  // 增益
  buffs: Buff[]

  // 状态
  isGolden: boolean
  hasAttacked: boolean

  // 横屏位置
  position: Position
}

interface Position {
  side: 'left' | 'right'
  row: 'front' | 'middle' | 'back'
  col: 1 | 2 | 3
}
```

## 精英随从

### 核心属性

| 属性 | 值 |
|------|-----|
| 星级 | 7 星 |
| 体型 | 2×2 方阵（占4格） |
| 获取方式 | 仅精英关卡/Boss关卡掉落 |
| 玩家商店 | 最高6级，无法购买 |
| 最多上场 | 1 个（6格限制） |

### 位置规则

- 可放置在任意空位（需满足 2×2 空间）
- 最多同时存在 1 个精英随从

### 光环计算

精英随从占据的 **4 个格子都参与** 同排/同列光环计算。

```
示例布局：
位置1 位置2 位置3
位置4 [精英] [精英]
位置5 [精英] [精英]

精英的4个位置都触发光环效果
```

### 代码实现注意事项

```typescript
// 检查是否可以放置精英随从
function canPlaceElite(position: Position, battlefield: (Follower | null)[]): boolean {
  const positions = getElitePositions(position)
  return positions.every(p => battlefield[p] === null)
}

// 光环计算时，精英的每个位置都计算
function calculateAura(aura: AuraEffect, battlefield: Follower[]): number {
  let count = 0
  for (const follower of battlefield) {
    if (follower.isElite) {
      count += 4
    } else {
      count += 1
    }
  }
  return count
}
```

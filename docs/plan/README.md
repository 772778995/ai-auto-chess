## 五层递进

| 层级 | 目标 | 预计工期 | 验收标准 |
|------|------|----------|----------|
| **01-基础设施层** | 建立正确的类型系统和数据结构 | 1-2周 | 类型通过编译，API可访问 |
| **02-核心玩法层** | 实现单人可体验的核心游戏循环 | 3-4周 | 单机可玩，商店战斗正常 |
| **03-系统集成层** | 实现8人在线对战 | 2-3周 | 8人匹配，状态同步正确 |
| **04-内容扩展层** | 添加完整游戏内容 | 4-6周 | 所有词条生效，内容丰富 |
| **05-体验优化层** | 提升用户体验和视觉表现 | 2-3周 | 动画流畅，新手易上手 |

## 核心机制变更

- **伤害机制**: 攻击方不再受到被攻击方的反伤（只有反击词条能反伤）
- **布局**: 横屏 3×2 战场站位
- **拼点**: 战斗前拼点决定先攻权

## 快速启动

建议从 **01-基础设施层** 开始，首先完成：

1. [类型定义](./01-基础设施层/01-类型定义.md) - 重构 game.ts
2. [数据文件](./01-基础设施层/02-数据文件.md) - 创建初始数据模板

## 技术栈

### 前端
- Vue 3 + TypeScript
- Pinia 状态管理
- Phaser 3 游戏引擎
- UnoCSS 样式系统

### 后端
- Hono.js Web框架
- Bun 运行时
- PostgreSQL 数据库
- WebSocket 实时通信

### 共享
- Zod 运行时验证
- msgpackr 二进制序列化
